# 苍穹外卖中的技术总结

### DTO、VO、POJO

#### DTO (Data Transfer Object)

**作用：在不同服务 / 接口 / 层之间传输数据**（比如前端→后端、服务 A→服务 B）。

核心场景

-   接收**前端传过来的参数**
-   微服务之间调用时传递数据
-   只包含**需要传输的字段**，不暴露敏感信息

```java
// 只接收前端传的账号和密码，不包含数据库字段
public class UserLoginDTO {
    private String username;
    private String password;
    // getter/setter
}
```



#### VO (View Object)

**作用：给前端页面展示数据**（Controller → 前端）。

核心场景

-   封装**返回给前端的最终数据**
-   可以组合多个表的数据
-   只保留页面需要的字段

```java
// 前端只需要展示这些，不需要密码、创建时间等
public class UserInfoVO {
    private Long userId;
    private String username;
    private String avatar;
    private Integer age;
    // getter/setter
}
```



#### POJO (Plain Old Java Object)

一种**统称、父概念**，不是具体的业务对象。

特点

-   只有**私有属性 + getter/setter**
-   无继承、无复杂逻辑
-   不依赖任何框架
-   **DTO、VO 本质上都是 POJO**

```java
// 标准POJO
public class User {
    private Long id;
    private String name;

    // getter、setter
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
}
```



### 后端接收前端参数的格式

在苍穹外卖项目中，后端中的 Controller 需要接收来自前端的请求。

前端的常见 HTTP 请求为：

*   GET：查询、获取数据
*   POST：新增、提交数据
*   PUT：修改、更新数据
*   DELETE：删除数据

对应的三种传参方式：

*   路径参数
*   Query 参数
*   JSON 参数

对应这四种请求和三种传参方式，后端接收参数的方式概括为如下表格：

| 请求方式   | 主要用途 | 前端常用传参格式 | 后端接收注解             | 苍穹外卖常见场景 |
| ---------- | -------- | ---------------- | ------------------------ | ---------------- |
| **GET**    | 查询数据 | 路径 `/user/100` | **@PathVariable**        | 根据 ID 查询     |
|            |          | Query `?name=xx` | **省略 / @RequestParam** | 分页、条件查询   |
| **POST**   | 新增数据 | JSON             | **@RequestBody**         | 新增员工、登录   |
| **PUT**    | 修改数据 | JSON             | **@RequestBody**         | 修改员工、起停售 |
| **DELETE** | 删除数据 | 路径 `/user/100` | **@PathVariable**        | 根据 ID 删除     |

具体代码实现：

（1）路径参数

形式：

URL：`/admin/employee/100`

参数直接写在 URL 路径里面，用 `{}` 占位，接收注解为 `@PathVariable`

```java
@GetMapping("/admin/employee/{id}")
public Result getById(@PathVariable Long id)
```

（2）Query 参数

形式：

URL：`/admin/employee/page?name=张三&page=1&pagesize=10`

URL 后面用 `?` 开始，GET、POST 请求都能用，接收注解为 `@RequestParam`（可不写，默认就是）

```java
@GetMapping("/admin/employee/page")  // 如果已经在类注解中已经写明 @RequestMapping("/admin/employee")，可以省略重复的部分
public Result page(String name, Integer page, Integer pageSize)
```

当然，也可以将 `String name`、`Integer page`、`Integer pageSize` 这三项封装为一个DTO，然后直接用这个 DTO 来接收参数：

```java
@GetMapping("/admin/employee/page")
public Result page(EmployeePageQueryDTO employeePageQueryDTO)
```

其中 `EmployeePageQueryDTO` 的定义为：

```java
package com.sky.dto;

import lombok.Data;
import java.io.Serializable;

@Data
public class EmployeePageQueryDTO implements Serializable {
    // 员工姓名
    private String name;
    // 页码
    private int page;
    // 每页显示记录数
    private int pageSize;
}
```

（3）JSON 参数（请求体参数）

主要用于 POST、PUT 请求，用于传递复杂参数，接收注解为：`@RequestBody`

```java
@PostMapping("/admin/employee")
public Result save(@RequestBody EmployeeDTO employeeDTO)
```
